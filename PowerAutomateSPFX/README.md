# power-automate-spfx

## Summary

This solution demonstrates how to **trigger a Power Automate flow from an SPFx web part** without executing it as "anyone." Instead, it enables triggering the flow using **app context** or **elevated permissions**, giving more control and security over workflow executions.

**Blog post:** [Execute Power Automate Workflow from SPFx (Not as Anyone)](https://clavinfernandes.wordpress.com/2025/11/16/execute-power-automate-workflow-from-spfx-not-as-anyone/)
